Better-cmd Installation Guide

This document explains how to install and uninstall Better-cmd by adding or removing the program's bin folder to/from your user PATH environment variable on Windows.

Installation Process

1. Automatic Installation

Run the installation script from Command Prompt or PowerShell:

cscript //nologo install.vbs

What happens:

  • The script detects the bin folder located next to it
  • Adds this folder to your user PATH if not already present
  • Shows confirmation message upon success
2. Automatic Uninstallation

To remove Better-cmd from your PATH:

cscript //nologo uninstall.vbs

What happens:

  • The script removes the bin folder from your user PATH
  • Shows confirmation message upon completion

Important Notes:

  • Only modifies your user PATH (no system-wide changes)
  • No administrator privileges required
  • Restart your terminal after installation/uninstallation
  • Scripts don't copy files - ensure bin folder exists

Manual Installation

1. Adding to PATH Manually
  1. Open Start Menu → Search for "Environment Variables" → Select "Edit the system environment variables"
  2. In System Properties, click "Environment Variables..."
  3. Under User variables, select Path → Click Edit
  4. Click New → Enter full path to the bin folder
  5. Click OK to save changes
  6. Restart your terminal
2. Removing from PATH Manually
  1. Follow steps 1-3 above to open PATH editor
  2. Select the bin folder entry → Click Delete
  3. Save changes and restart terminal

Advanced Configuration

Changing Installation Location

To move Better-cmd to a different location:

  1. Move the entire folder containing bin to new location
  2. Remove old path (using uninstall script or manually)
  3. Add new path (using install script or manually)
  4. Restart terminal

Note: The VBScript expects the bin folder to be adjacent to it. Moving folders separately may require manual PATH updates.

Troubleshooting

Issue Solution
Script execution blocked Check Windows Script Host is enabled or run as Administrator
Commands not recognized
  • Restart terminal
  • Check PATH with echo %PATH%
  • Verify executables exist in bin
Multiple versions PATH order determines which version runs - keep only one
